# V0.5 RAG verbeter-bundel — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Een gebundelde V0.5 RAG kwaliteitsslag op v0.4 die de bot menselijker laat klinken, algemene-kennis-vragen binnen het domein opvangt, claim-verification daadwerkelijk gebruikt om hallucinaties te corrigeren, en de eval-pipeline eerlijk maakt door parentExcerpt te tonen aan de judge.

**Architecture:** Append-only nieuwe `V0_5` BotConfig met drie nieuwe gedrag-flags (general-knowledge router, claim-regenerate, soft word-ban). Backend krijgt een tweede-stage re-classifier in `lib/v0/server/reclassify.ts` die alleen draait bij zero-hits retrieval. Stream-replacement event in de SSE/NDJSON-laag vervangt het zichtbare antwoord bij low-verified-ratio. parentExcerpt propageert van retrieval naar judge én sources-UI; oude versies blijven onaangetast.

**Tech Stack:** Next.js 16.2 App Router, TypeScript, OpenAI SDK (gpt-4o-mini/gpt-4o + text-embedding-3-small), Supabase Postgres + pgvector, Playwright e2e, tsx scripts.

```ts
function toSource(c: RetrievedChunk): ChatSource {
  const contentExcerpt =
    c.content.length > EXCERPT_CHARS
      ? c.content.slice(0, EXCERPT_CHARS).trimEnd() + '…'
      : c.content;
  // Parent-content is alleen aanwezig als bot.parentDocumentRetrieval=true
  // EN de chunk een gehydrateerde parent had. In de huidige codebase wordt
  // parent_content op de RetrievedChunk gezet door retrieveChunks(withParents)
  // / retrieveChunksHybrid(withParents) of door hydrateParentContent() (zie
  // regel ~768). Null of undefined → we slaan parentExcerpt over zodat oude
  // responses (zonder dit veld) backward-compat blijven.
  let parentExcerpt: string | null | undefined = undefined;
  if (typeof c.parent_content === 'string' && c.parent_content.length > 0) {
    parentExcerpt =
      c.parent_content.length > PARENT_EXCERPT_CHARS
        ? c.parent_content.slice(0, PARENT_EXCERPT_CHARS).trimEnd() + '…'
        : c.parent_content;
  } else if (c.parent_content === null) {
    // Hydratie geprobeerd maar gefaald → expliciet null zodat de UI weet:
    // "er was een parent, maar we konden hem niet ophalen". Onderscheid van
    // "geen parent_chunk_id" (undefined).
    parentExcerpt = null;
  }
  return {
    id: c.id,
    filename: c.filename,
    similarity: c.similarity,
    contentExcerpt,
    ...(parentExcerpt !== undefined ? { parentExcerpt } : {}),
  };
}
```

## Task 3: Cascade-kosten centraliseren via MODEL_COSTS

**Files:**
- Modify: `lib/v0/server/rag.ts:1567-1571` (hardcoded GPT4O_IN/OUT)
- Modify: `lib/ai/llm.ts:38-44` (MODEL_COSTS — extend met USD context)

**Rationale:** De spec zegt: "Import `MODEL_COSTS` uit `lib/ai/llm.ts`, kijk model-naam op runtime op." Echter — `MODEL_COSTS` in `lib/ai/llm.ts:38-44` is gedefinieerd in **EUR** (`cost_eur`-context), niet in USD. De rest van V0 rekent expliciet in USD (zie `CHAT_INPUT_PER_M_USD` op regel 33, `JUDGE_INPUT_PER_M_USD` in eval.ts:33). We mogen niet zomaar EUR-rates in USD-aggregates stoppen — dat zou de bot_cost_usd kolom in query_log corrupt maken.

**Beslissing:** We splitsen het probleem in twee:

1. We hernoemen niet en muteren niet `MODEL_COSTS`. In plaats daarvan voegen we naast `MODEL_COSTS` een tweede export toe — `MODEL_COSTS_USD` — die exact dezelfde shape heeft maar de gepubliceerde USD-rates bevat. Beide bestaan side-by-side; `MODEL_COSTS` blijft voor V1 (EUR-billing), `MODEL_COSTS_USD` is wat V0 nodig heeft.
2. In `rag.ts` vervangen we de hardcoded GPT4O_IN/OUT door een lookup in `MODEL_COSTS_USD` met fallback.

- [ ] **Step 1: Uitbreiden `lib/ai/llm.ts` met MODEL_COSTS_USD**

In `lib/ai/llm.ts`, na de bestaande `MODEL_COSTS` (regel 38-44), voeg toe:

```ts
/**
 * Per-million-token costs in USD — pure provider-rate tabel. Gebruikt door V0
 * waar token-kosten in USD opgeteld worden (query_log.cost_usd is USD).
 *
 * Houd in sync met provider pricing pages. Bij prijswijziging: pas hier aan
 * en de wijziging propageert automatisch naar elke aanroeper. NIET hardcoden
 * in callsites.
 *
 * V1 callers gebruiken MODEL_COSTS (EUR) voor billing; V0 callers gebruiken
 * MODEL_COSTS_USD voor cost-discipline-telemetrie.
 */
export const MODEL_COSTS_USD = {
  'claude-haiku-4-5':   { input_per_m: 1.0,  output_per_m: 5.0 },
  'claude-sonnet-4-6':  { input_per_m: 3.0,  output_per_m: 15.0 },
  'gpt-4o-mini':        { input_per_m: 0.15, output_per_m: 0.60 },
  'gpt-4o':             { input_per_m: 2.50, output_per_m: 10.0 },
} as const satisfies Record<string, { input_per_m: number; output_per_m: number }>;

export type SupportedModelUsd = keyof typeof MODEL_COSTS_USD;

/**
 * Lookup-helper. Onbekend model → 0/0 (neutrale fallback ipv crash). Cost-
 * onderschatting bij een onbekend model is acceptabel voor V0; we loggen
 * de hit via console.warn zodat de mismatch zichtbaar wordt.
 */
export function costForModelUsd(
  model: string,
  inputTokens: number,
  outputTokens: number,
): number {
  const rates = (MODEL_COSTS_USD as Record<string, { input_per_m: number; output_per_m: number }>)[model];
  if (!rates) {
    console.warn(`[MODEL_COSTS_USD] onbekend model: ${model} — cost berekend als 0`);
    return 0;
  }
  return (
    (inputTokens / 1_000_000) * rates.input_per_m +
    (outputTokens / 1_000_000) * rates.output_per_m
  );
}
```

function parseReclassifyOutput(raw: string): ReclassifyResult | null {
  const t = raw.trim().toUpperCase();
  // Tolerant voor trailing text — pak alleen het eerste woord-achtige token.
  const first = t.split(/\s+/)[0]?.replace(/[.,!?:;]+$/, '');
  if (first === 'GENERAL') return 'general';
  if (first === 'OFF_TOPIC' || first === 'OFFTOPIC' || first === 'OFF-TOPIC') return 'off_topic';
  if (first === 'FALLBACK') return 'fallback';
  return null;
}

/**
 * Tweede-stage classifier. Niet aan te roepen vanuit de pre-processor zelf
 * (die heeft geen retrieval-info); alleen vanuit runRagQueryStreaming bij
 * zero-hit retrieval EN bot.generalKnowledgeEnabled.
 *
 * Cost: één LLM-call met bot.chatModel (gpt-4o-mini default) ~$0.0001.
 * Faalt veilig: bij API/parse-error retourneert 'fallback' zodat de bestaande
 * FALLBACK_MESSAGE-flow draait (= geen LLM-call voor het antwoord).
 */
export async function reclassifyAfterZeroHits(
  question: string,
  bot: BotConfig,
): Promise<ReclassifyOutput> {
  const fallbackResult: ReclassifyOutput = {
    category: 'fallback',
    inputTokens: 0,
    outputTokens: 0,
    costUsd: 0,
  };
  try {
    const resp = await openai().chat.completions.create({
      model: bot.chatModel,
      temperature: 0.0,
      max_tokens: 10,
      messages: [
        { role: 'system', content: RECLASSIFY_SYSTEM },
        { role: 'user', content: question },
      ],
    });
    const text = resp.choices[0]?.message?.content ?? '';
    const inputTokens = resp.usage?.prompt_tokens ?? 0;
    const outputTokens = resp.usage?.completion_tokens ?? 0;
    // gpt-4o-mini rates: $0.15/1M in, $0.60/1M out — komt overeen met
    // CHAT_INPUT_PER_M_USD/OUTPUT_PER_M_USD in rag.ts. Hier hardcoden om
    // circular import te vermijden; bij model-wijziging update beide
    // locaties (V1 Phase 4 lost dit op via een gedeelde lib/ai/llm.ts).
    const CHAT_INPUT_PER_M_USD = 0.15;
    const CHAT_OUTPUT_PER_M_USD = 0.6;
    const costUsd =
      (inputTokens / 1_000_000) * CHAT_INPUT_PER_M_USD +
      (outputTokens / 1_000_000) * CHAT_OUTPUT_PER_M_USD;
    const parsed = parseReclassifyOutput(text);
    if (!parsed) {
      console.warn(`[reclassify] parse-error op output: ${text.slice(0, 80)}`);
      return { ...fallbackResult, inputTokens, outputTokens, costUsd };
    }
    return { category: parsed, inputTokens, outputTokens, costUsd };
  } catch (err) {
    console.warn(
      '[reclassify] API error:',
      err instanceof Error ? err.message : err,
    );
    return fallbackResult;
  }
}
