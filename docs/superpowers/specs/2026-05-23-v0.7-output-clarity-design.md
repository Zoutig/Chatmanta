# v0.7 — output-clarity: scherpere lengtes + structuur in de widget

**Status:** design, niet geïmplementeerd
**Datum:** 2026-05-23
**Eigenaar:** Sebastiaan
**Trigger:** widget-output voelt te wollig op "Kort", "Normaal" is onnodig opgeblazen met vulzinnen, "Lang" mist structuur. Bezoeker scant niet — leest een wall of text.

## Doel

Drie verbeteringen samen, één nieuwe bot-versie (`v0.7`):

1. **Strakkere lengte-definities** in de system-prompt — kort écht kort, normaal adaptief, lang gestructureerd.
2. **BLUF + anti-preamble + anti-meta-talk** als gedragsregels in de prompt.
3. **Widget-renderer ondersteunt bullets, witregels en sub-koppen** zodat de prompt-uitvoer ook visueel als gestructureerde output landt.

Niet in scope: volledige markdown (eigen heading-syntaxis, tabellen, hyperlinks-styling).

## Wat verandert

### 1. Lengte-strings — versie-bewust

`lib/v0/style.ts` mag de bestaande `LENGTH_INSTRUCTION` niet stilzwijgend overschrijven (zou v0.1–v0.6 evals invalideren). In plaats daarvan:

```ts
const LENGTH_INSTRUCTION_V1: Record<Length, string> = { /* huidig */ };
const LENGTH_INSTRUCTION_V2: Record<Length, string> = {
  short:
    'Houd het ULTRA-kort: 1 zin als het kan, maximaal 2. Geen volzinnen waar komma\'s genoeg zijn. Geen aanloop of slot.',
  medium:
    'Geef het minimum dat compleet is — zo kort als de vraag toelaat, zo lang als nodig om volledig te zijn. Bij een simpel feit: 1-2 zinnen. Bij meerdere onderdelen of een vergelijking: paragraafje. Geen verplichte minimum-lengte, geen vulling.',
  detailed:
    'Geef het volledige antwoord met structuur: paragrafen met witregels, opsommingen waar er 3+ parallelle items zijn, en gebruik **vetgedrukte koppen** voor sub-onderwerpen (bv. "**Openingstijden**" gevolgd door details). "Meer structuur", niet "meer woorden" — voeg geen vulling toe voor de schijn van diepgang.',
};
```

`buildSystemPrompt()` krijgt een derde optionele parameter `outputStyleVersion: 'v1' | 'v2' = 'v1'` die kiest welke map. Default `'v1'` zodat alle bestaande aanroepen ongewijzigd blijven.

### 2. BLUF + anti-preamble — als prompt-block in v0.7

Nieuw blok in `bots.ts`:

```ts
const V0_7_OUTPUT_RULES_BLOCK = `

OUTPUT-DISCIPLINE:

LEAD MET HET ANTWOORD (BLUF):
- Eerste zin = direct antwoord op de vraag. Geen aanloop, geen herhaling van de vraag.
- Ja/nee-vragen: woord 1 is "Ja" of "Nee". Dan pas toelichting.

GEEN PREAMBLE:
- VERBODEN openings-formuleringen: "Bedankt voor je vraag", "Goeie vraag", "Leuk dat je het vraagt", "Zoals je vroeg", "Wat betreft je vraag", "Op basis van de beschikbare informatie".
- VERBODEN als slot: een conclusie-zin die alles herhaalt ("Kortom:...", "Samenvattend:..."). Stop zodra de vraag is beantwoord.

GEEN OPGEBLAZEN ZINNEN:
- Verzin geen bufferinformatie ("we proberen binnen 24u te reageren" — alleen als dat letterlijk in de bronnen staat).
- Geen herhaling van wat de gebruiker net zei.
- Geen meta-talk over wat je gaat doen ("Ik zal je uitleggen dat..."). Doe het gewoon.

`;
```

Toegevoegd aan `V0_7.systemPrompt` (niet aan v0.6) zodat bestaande versies onveranderd blijven.

### 3. v0.7 BotConfig

```ts
const V0_7: BotConfig = {
  ...V0_6,
  version: 'v0.7',
  label: 'v0.7 — output-clarity',
  description:
    'v0.6 plus scherpere lengte-prompts (kort=1-2 zinnen, normaal=adaptief, lang=gestructureerd), BLUF-lead, anti-preamble, en bullets/witregels renderen nu in de widget.',
  outputStyleVersion: 'v2',
  systemPrompt: V0_6.systemPrompt + V0_7_OUTPUT_RULES_BLOCK,
};
```

`outputStyleVersion?: 'v1' | 'v2'` wordt toegevoegd aan `BotConfig`. Default ondefined ≡ `'v1'`.

Bot-registry update:

```ts
export const BOTS = { ..., [V0_7.version]: V0_7 };
export const LATEST_BOT_VERSION = V0_7.version;
export const BOT_VERSIONS_ORDERED = [..., V0_7.version];
```

`EVAL_DEFAULT_VERSIONS` blijft automatisch correct (laatste twee = v0.6 + v0.7).

### 4. RAG-pipeline doorgeven van outputStyleVersion

`lib/v0/server/rag.ts` roept `buildSystemPrompt(baseSystem, style)` aan. Wijzig naar `buildSystemPrompt(baseSystem, style, bot.outputStyleVersion)`. Eén regel.

### 5. Klantendashboard prompt-preview

`app/components/prompt-view.tsx` rendert dezelfde `buildSystemPrompt`. Moet ook de `outputStyleVersion` van de actieve bot doorgeven anders drift de preview. Onderzoek waar deze component aangeroepen wordt en pass de bot-config door.

### 6. Widget-renderer upgrade

`renderMarkdownLite()` in `app/widget/components/chatmanta-widget.tsx` ondersteunt nu alleen `**bold**`. Uitbreiden met:

**Bullets** — een regel die met `- ` of `* ` begint wordt een `<li>`. Opeenvolgende bullet-regels samenvouwen tot één `<ul>`. Geen nested lists (bot wordt niet geinstrueerd ze te maken).

**Witregels** — een lege regel tussen twee blokken wordt een paragraph-break (rendert als `<div style={{marginTop: 8}}>` of `<br>`). Géén full paragraph-tag wrapping; de huidige `whiteSpace: 'pre-wrap'` doet de rest.

**Sub-koppen** komen gratis mee: een regel als `**Openingstijden**` gevolgd door witregel = vet kopje + witruimte. Geen aparte heading-syntaxis.

Niet ondersteund: genummerde lijsten (`1.`), nested bullets, headings met `#`, tabellen, links. Als de bot per ongeluk zoiets stuurt valt het terug op plain text — niet crashen.

Approx 40-70 regels code voor de upgrade. Bestaande `lib/widget/render-markdown.tsx` (full md) is geen optie: te zwaar voor bubble-formaat (h1 met border-bottom, etc.).

## Risico's en mitigaties

| Risico | Mitigatie |
|---|---|
| Eval-regressie v0.7 vs v0.6 op compleetheid | Eerst eval-run `eval:run -- --versions=v0.6,v0.7` voor we live gaan. Memory `feedback_eval_cost_discipline` — alleen 2 versies, niet hele set. |
| Bot interpreteert "kort = 1 zin" te strikt en mist info | Eval-judge dimensie `answer_length_appropriate` ("too_curt") is al in eval-pipeline. Als too_curt-rate op v0.7 > 5% boven v0.6: lengte-string opnieuw tunen. |
| Oude bot-versies sturen plain text die nu raar rendert | Bullet-detectie alleen op `^- ` of `^\* ` aan regelbegin — onveranderd plain-text gedraagt zich identiek. Smoke-test op v0.5/v0.6 transcripten. |
| Anti-preamble vangt te veel ("Ja" begint met goed nieuws-toon) | "Geen preamble" verbiedt aanloop, niet bevestigend antwoorden. Voorbeelden in prompt verduidelijken bedoeling. |
| Prompt-view drift in klantendashboard | Pass `outputStyleVersion` mee in elke call site van `buildSystemPrompt`. Parameter blijft optioneel met `'v1'`-default zodat bestaande aanroepen niet stuk gaan; alleen `rag.ts` en `prompt-view.tsx` hoeven aangepast. |

## Versie-strategie

**Nieuwe v0.7** in append-only stijl. De `feat/seb/v07-prio-1234`-branch (cost/latency-experiment, niet productie-waardig per memory) krijgt geen merge meer — die branch kan worden afgesloten of hernoemd naar `feat/seb/v07-cost-experiment` zodat de v0.7-naam vrij is.

**Alternatief overwogen, afgewezen:** in-place patch op v0.6 (zoals de bridging-block precedent). Afgewezen omdat dit een user-facing output-style change is die we A/B willen kunnen meten — een nieuwe versie maakt evals zinvol.
