## Wat Run 1 vs Run 2 onthulde

**Run 1** had `CONCURRENCY=5` en `claimRegenerateThreshold=0.5`:
- 25 judge-parse failures (rate-limit hits op gpt-4o TPM 30k/min)
- v0.5 leek 0.34 onder v0.4 — voornamelijk meet-artefact

**Run 2** met `CONCURRENCY=2` en `claimRegenerateThreshold=0.3`:
- **0 judge-parse failures** — concurrency-tweak loste rate-limit issue 100% op
- v0.5 (3.03) lichtjes boven v0.4 (2.89)
- Schoon signaal voor de PR

### Twee belangrijke side-findings

1. **Judge-variance is hoog tussen runs.** Vergelijk v0.4 tussen Run 1 (3.38) en Run 2 (2.89) — een verschil van 0.49 punten op exact dezelfde data. v0.2 zelfs 0.85. De gpt-4o-judge geeft niet-deterministische scores. Implicatie: een verschil van <0.5 tussen versies is statistisch insignificant met deze N=25. Voor latere versies: meerdere judge-runs middelen, of N verhogen.

2. **Concurrency in batch-scripts ≠ productie concurrency.** De `CONCURRENCY=5` in `scripts/v0-eval-run.ts` was alleen voor eval-batch. Productie (V1) handelt elke user-request parallel af zonder dat limiet — 10 gelijktijdige gebruikers wachten gewoon ~4-8s elk.

## Wat v0.5 wél verbeterde (kwalitatief, niet door scores)

- **Grounding-meting is eerlijker** — parentExcerpt in judge → judge ziet wat de LLM zag (~800 chars parent), niet meer alleen de match-chunk (~240 chars small)
- **GENERAL-pad werkt** — Playwright bevestigt: "Wat zijn MKB-bedrijven?" → disclaimer-antwoord met "Even kort: dit valt buiten onze specifieke documentatie, maar in het algemeen..." + sluitings-uitnodiging
- **OFF_TOPIC-pad werkt** — Playwright bevestigt: "Schrijf een gedicht over zalmen" → polite refusal, geen poëzie-output
- **Soft word-ban** — "document/bron/context" niet meer hardgeblokkeerd, natuurlijke nuance toegestaan; judge meet `meta_talk_present` als regressie-flag
- **Claim-regenerate** — alleen bij <30% verified-ratio (post-tune van 50%), niet meer overbodig getriggered

## Wat v0.5 (mogelijk) niet verbeterde — voor v0.6

- Eén echt-regressie-case in Run 1: een v0.4 smalltalk (992ms, score 5/5/5) werd in v0.5 een answer-poging (8372ms, score 0/0/0). Mogelijk een conversational query die door de strikte v0.5-SMALLTALK-prompt niet meer als smalltalk werd herkend. In Run 2 niet meer geverifieerd — kan random variance zijn, maar het bevestigen vereist een derde run of een specifieke regressie-test
- Judge-variance van 0.3-0.85 punten tussen runs op zelfde data — eval-infra-werk voor v0.6 (mock judge / batch judge / hogere N / meerdere runs middelen)
